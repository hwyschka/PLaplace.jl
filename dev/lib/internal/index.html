Static Data

PLaplace.StaticData — Type
mutable struct StaticData

Stores all data that is static throughout iteration of the interior-point method. Is supposed to be generated by corresponding constructor.

Fields

  • p::Float64: PDE parameter

  • d::Int8: Domain dimension

  • qdim::Int64: Image dimension, i.e. number of components

  • n::Int64: Number of nodes in domain mesh

  • m::Int64: Number of elements in domain mesh

  • calculation_nodes::Set{Int64}: Set of actually calculated nodes

  • nCalc::Int64: Number of calculated nodes

  • lengthu::Int64: Number of calculated FEM coefficients

  • c::AbstractVector{Float64}: System vector for convex optimization problem

  • R::Float64: Upper bound for auxilliary variable

  • omega::AbstractVector{Float64}: Vector of element weights

  • D::AbstractDict{Tuple{Int64, Int64}, SparseArrays.SparseMatrixCSC{Float64, Int64}}: Discrete derivative matrix

  • g::AbstractVector{Float64}: Prolonged boundary

  • b::AbstractDict{Tuple{Int64, Int64}, AbstractVector{Float64}}: Discrete derivative of the prolonged boundary condition, i.e. Dg

  • sigma::Int64: Constant for barrier self-concordance order

  • alpha::Float64: Constant factor for barrier terms

  • beta::Float64: Algorithm constant given by Nesterov as 1/9

  • gamma::Float64: Algorithm constant given by Nesterov as 5/36

  • tolInv::Float64: Inverse tolerance used for main path-following termination

  • tolFactor::Float64: Accuracy to tolerance factor

  • maxIter::Int64: Maximal number of iterations in path-following schemes

  • kappa::Float64: Initial stepsize for long and adaptive path-following

  • kappa_updates::Array{Int64, N} where N: Ranges of slow steps for update of kappa in adaptive path-following

  • kappa_powers::Array{Float64, N} where N: Powers in the update that is applied to kappa in the relevant ranges

  • maxIterationsBacktracking::Int64: Maximal number of iterations for backtracking in long and adaptive path-following

  • decrementFactorBacktracking::Float64: Decrement factor for backtracking in long and adaptive path-following

  • solveLS::Function: Function pointer on solving routine

  • factorize::Function: Funciton pointer on factorization routine

  • computePreconditioner::Function: Function pointer on preconditioner

PLaplace.BarrierFunction — Type
BarrierFunction

Object holdigng the relevant functions for evaluating a barrier function. Includes gradient and hessian as well as a specific initial guess.

Is not intended to be initialized directly but via sub-constructors corresponding to specific barrier files. See their documentations belov to identify the behaviour of specific barrier implemtations.

Fields

  • isadmissible::Function: Returns boolean value wether $x$ is in the corresponding admissible set.

  • value::Function: Returns float value of the barrier function at $x$.

  • gradient::Function: Returns vector-valued gradient of the barrier function at $x$.

  • hessian::Function: Returns matrix-valued Hessian of the barrier function at $x$.

  • initialguess::Function: Returns inital vector guess for starting an auxilliary path-following.

PLaplace.FiniteBarrier — Method
FiniteBarrier() -> PLaplace.BarrierFunction

Returns BarrierFunction object corresponding to default barrier function for finite p.

In this setting the admissible set is given by

\[ \mathcal{Q}_p = \left\{ (u,s) \in \mathbb{R}^n \times \mathbb{R}^m \, : \, s_i \geq \left(\sum_{j=1}^{d} \sum_{r=1}^{d'} [D^{(j,r)}(u+g)]_i^2 \right)^{\frac{p}{2}} \land \omega_i s_i \leq R \right\}\]

Defining the variables

\[ z_i(u,s) = s^{\frac{2}{p}} - \sum\limits_{j=1}^d \sum\limits_{r=1}^d [\underbrace{D^{(j,r)}(u+g)}_{=: y^{(j,r)}(u)}]_i^2 \quad \text{and} \quad \tau_i(s) = R -\omega_i s_i\]

it can be equipped with the self-concordant barrier

\[ F(x) = F(u,s) = -\sum\limits_{i=1}^{m} \log(z_i) -\sum\limits_{i=1}^{m} \log(s_i) -\sum\limits_{i=1}^{m} \log(\tau_i).\]

The derivatives

\[ F'(x) = \begin{bmatrix} F_u \\ F_s \end{bmatrix} \text{ and } F''(x) = \begin{bmatrix} F_{uu} & F_{us} \\ F_{us}^{\intercal} & F_{ss} \end{bmatrix}\]

are then given by

\[ \begin{aligned} F_{u} &=2 \sum_{j=1}^{d} \sum_{r=1}^{d'} [D^{(j,r)}]^{\intercal} \frac{y^{(j,r)}}{z} \\ F_{s} &= -\frac{2}{p}\frac{1}{z}s^{\frac{2}{p}-1} + \frac{\omega}{\tau}\\ F_{uu} &= 2 \sum_{j=1}^{d} \sum_{r=1}^{d'} [D^{(j,r)}]^{\intercal} Z^{-1} D^{(j,r)} \\ &\quad+ 4 \sum_{j_1=1}^{d} \sum_{r_1=1}^{d'}\sum_{j_2=1}^{d} \sum_{r_2=1}^{d'} (Y^{(j_1,r_1)}D^{(j_1,r_1)})^{\intercal} Z^{-1} D^{(j)} (Y^{(j_2,r_2)}D^{(j_2,r_2)}),\\ F_{us} &= -\frac{4}{p}\sum_{j=1}^{d} \sum_{r=1}^{d'} (Y^{(j,r)}D^{(j,r)})^{\intercal} Z^{-2} S^{\frac{2}{p}-1},\\ F_{ss} &= -\frac{2}{p} \left(\frac{2}{p} - 1 \right) Z^{-1} S^{\frac{2}{p}-2} + \frac{4}{p^2} Z^{-2} S^{\frac{4}{p}-2} + W^2 T^{-2},\\ \text{where } S &= \mathrm{diag}(s),\, W = \mathrm{diag}(\omega),\, Y = \mathrm{diag}(y),\, Z = \mathrm{diag}(z) \text{ and } T = \mathrm{diag}(\tau). \end{aligned}\]

An initial guess inside $\mathcal{Q}_{\infty}$ is given by

\[ \hat{x} = \begin{bmatrix} 0 \\ \hat{s} \end{bmatrix} \in \mathbb{R}^{n \times m}\]

where

\[ \hat{s} = 1 + \left(\sum_{j=1}^{d} \sum_{r=1}^{d'} [D^{(j,r)} g]_i^2 \right)^{\frac{p}{2}} \in \mathbb{R}^m.\]

PLaplace.AssemblyTracker — Type
mutable struct AssemblyTracker

Tracks if factorization or preconditioner got changed during an assembly of the IterationData.

Also tracks condition of system of system matrix, i.e. the barrier Hessian. Has to be tracked via this tool because after the assembly the matrix is usually only available as factorization.

  • singularity::Bool: Flag if hessian is singular.

  • factorization::Bool: Flag if factorization got changed.

  • preconditioner::Bool: Flag if precondtioner got changed.

    I::PLaplace.IterationData,
    A::PLaplace.AlgorithmData,
    S::PLaplace.StaticData,
    L::PLaplace.LogData
) -> Int64

Executes auxilliary path-following with adaptive stepsize.

The iteration is performed on the IterationData, which will later store the final iterate as well as all the corresponding barrier terms. The number of required iterations and potential messages will be stored in AlgorithmData. If LogData is verbose, data per iteration will be written to the output stream. Further, if a file is provided, the log will be also exported to that file.

pathfollowing_main_adaptive!(
    I::PLaplace.IterationData,
    A::PLaplace.AlgorithmData,
    S::PLaplace.StaticData,
    L::PLaplace.LogData
) -> Int64

Executes main path-following with adaptive stepsize.

The iteration is performed on the IterationData, which will later store the final result. If the iteration did not converge, the last iterate will still be provided as a result and the obtained accuracy stored in AlgorithmData. The number of required iterations and potential messages will be stored in AlgorithmData in any case. If LogData is verbose, data per iteration will be written to the output stream. Further, if a file is provided, the log will be also exported to that file.
